03

User Flow

The complete journey from login to payout, including all API touchpoints.

1
User Login ‚Üí Eligibility Pre-fetch
When a user logs in, we fetch their personalized survey availability from MC.
  • Get user's profile data (DOB ‚Üí age, gender)
  • Fetch active bids from our survey_inventory table
  • Call POST /supplier/eligibility with qualifications + bid IDs
  • Filter to bids with CPI ‚â• $2.00
  • Store in user_survey_eligibility with timestamp
2
Feed Display ‚Üí Survey Card
The mission feed shows a survey card based on cached eligibility.
  • If eligible surveys exist: Show active card with "Earn $1 or more"
  • If no eligible surveys: Show disabled "Check back later" state
3
Card Tap ‚Üí Re-fetch & Modal
When user taps the card, check freshness and show the payout modal.
  • Check fetched_at timestamp
  • If older than 60 seconds, re-fetch from MC
  • Calculate user payout: (CPI √ó 50% / 100)
  • Calculate time: LOI / 60 ‚Üí minutes
  • Display modal: "Your Payout: $X.XX" + "~Y min"
4
Start Survey ‚Üí Entry URL
Build and sign the MC entry URL, open in new tab.
  • Create survey_sessions record with status='pending'
  • Build entry URL with required params
  • Sign URL with Ed25519 private key
  • Open in new browser tab
// Entry URL Format https://surveys.insights-today.com/v1/survey/entry ?bid_id={bid_id} &panelist_id={user_id} &supplier_id={supplier_id} &session_metadata={our_session_id} &qualifications=age,{age},gender,{gender_code} &signature={ed25519_signature}
5
Survey Completion ‚Üí Redirect
MC redirects back to our page with status and payout info.
  • Verify MC's signature on redirect URL
  • Look up session by session_metadata
  • Update status, actual_payout, completed_at
  • If complete: Credit user wallet
  • Display appropriate result screen

# Action Endpoint
1 Register our public key POST /user/public-keys
2 Set default redirect URL PUT /user/redirect-urls/default
3 Set status-specific redirects PUT /user/redirect-urls/{status_id}
4 Get MC's public key (for verification) GET /lookup/public-key
